## ⚖️ ประเด็นที่อยากให้ทนายพิจารณา/ตัดสินใจ

1. **บทบาท controller/processor** — สำหรับข้อมูลผู้เช่า เราวางให้ *เจ้าของหอ = ผู้ควบคุม* และ *Chao-Dee = ผู้ประมวลผล* (มี DPA รองรับ) — ถูกต้องเหมาะสมหรือไม่?
2. **เลขบัตรประชาชนผู้เช่า** — เก็บผ่านระบบ ควรมีข้อความ/ฐานกฎหมายเพิ่มหรือไม่ (ไม่ใช่ข้อมูลอ่อนไหวตาม ม.26 แต่มีความเสี่ยงสูง)
3. **การโอนข้อมูลข้ามประเทศ** — Supabase/Vercel อยู่ต่างประเทศ (คาดว่าสิงคโปร์) เราแจ้งตาม ม.28/29 — ต้องมีสัญญา/มาตรการเพิ่มหรือไม่ และควร pin region ไหน
4. **DPO** — กิจกรรมเข้าข่ายต้องแต่งตั้ง DPO ตาม ม.41 หรือไม่ (แพลตฟอร์มเก็บข้อมูลผู้เช่าจำนวนมาก)
5. **ระยะเวลาเก็บข้อมูล** — เราระบุเอกสารภาษี ≥ 5 ปี ส่วนอื่นลบเมื่อปิดบัญชี — เหมาะสมหรือควรกำหนดละเอียดกว่านี้
6. **ข้อคืนเงิน** (Terms ข้อ 3) — เราปรับให้ไม่เป็นข้อสัญญาไม่เป็นธรรม (พ.ร.บ.ข้อสัญญาไม่เป็นธรรม 2540) — เพียงพอหรือไม่
7. **เงื่อนไข marketplace** (Terms ข้อ 7) — แพลตฟอร์มเป็นตัวกลาง ไม่ใช่คู่สัญญาเช่า/นายหน้า — ครอบคลุมความรับผิดพอหรือไม่
8. **VAT/ใบกำกับภาษี** — ปัจจุบัน**ยังไม่จด VAT** จึงไม่ออกใบกำกับภาษี (ระบบล็อกไว้) — ยืนยันแนวทาง และเงื่อนไขที่ต้องจด (รายได้เกิน 1.8 ลบ./ปี)
9. **DBD e-commerce** — ต้องจดทะเบียนพาณิชย์อิเล็กทรอนิกส์ (ธุรกิจขาย/ให้บริการออนไลน์ + marketplace) ภายใน 30 วัน + แสดงเครื่องหมาย DBD Registered + เลขทะเบียนบนเว็บ — โทษถ้าไม่จด: ปรับไม่เกิน 2,000 บาท + วันละไม่เกิน 100 บาท
10. **⚠️ วัตถุประสงค์ของห้างฯ ไม่ตรงกับธุรกิจ (ยืนยันแล้ว)** — จากข้อมูล DBD/Open Data (dataforthai) ห้างฯ จดวัตถุประสงค์ไว้เพียง **"ให้บริการจัดงานเลี้ยง" (หมวดการบริการด้านการจัดเลี้ยง)** เท่านั้น **ไม่ครอบคลุมซอฟต์แวร์/SaaS/แพลตฟอร์มออนไลน์** ที่ประกอบการจริง → ควรจดเพิ่มวัตถุประสงค์ก่อน/พร้อมกับจด e-commerce (ดูร่างวัตถุประสงค์ที่ [dbd-objectives-draft.md](dbd-objectives-draft.md))

## สถานะปัจจุบัน (เชิงเทคนิค)
- ⚠️ **วัตถุประสงค์ห้างฯ = "จัดเลี้ยง" เท่านั้น** (ยืนยันจาก Open Data) — ต้องจดเพิ่มให้ครอบคลุมซอฟต์แวร์/marketplace
- ⏳ DBD e-commerce — รอยืนยันว่าจดแล้วหรือยัง (ถ้าจดแล้วให้ส่งเลขทะเบียนมาใส่ footer; ถ้ายัง ต้องจดภายใน 30 วัน)
- ⏸️ VAT ยังไม่จด — ระบบล็อกการออกใบกำกับภาษีไว้ถูกต้อง
- ✅ มาตรการความปลอดภัย: RLS แยกองค์กร, Storage แยกหอ, rate-limit, audit log, secret ฝั่ง server
