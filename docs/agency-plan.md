# แผนระบบเอเจนซี่หาผู้เช่า (ค่านายหน้า 1 เดือน)

เอกสารวางแผน — ยังไม่ได้ลงมือพัฒนา ใช้เพื่อตัดสินใจ + ส่งต่อทนาย/บัญชี

---

## 1. โมเดลธุรกิจ — ต้องเลือกก่อน ⚠️

| แบบ | คำอธิบาย | ข้อดี | ข้อเสีย |
|---|---|---|---|
| **A. Chao-Dee เป็นเอเจนซี่เอง** (แนะนำเริ่มที่นี่) | ใช้ marketplace /rent ที่มีอยู่หาผู้เช่าให้เจ้าของหอ แล้วเก็บค่านายหน้า | ต่อยอดของเดิม (มี lead อยู่แล้ว) · ไม่ต้องสร้างระบบเอเจนต์ · คุมคุณภาพได้ | ต้องมีคนทำงานจริง (นัดดู/ปิดดีล) |
| **B. เปิดให้เอเจนต์ภายนอกสมัคร** | เอเจนต์แนะนำผู้เช่า ได้ส่วนแบ่ง แพลตฟอร์มหัก % | ขยายเร็ว ไม่ต้องลงแรงเอง | ต้องสร้างระบบเอเจนต์+จ่ายเงิน+ภาษี · คุมคุณภาพยาก |
| **C. ผสม** | เริ่ม A แล้วเปิด B ทีหลัง | ยืดหยุ่น | ซับซ้อนขึ้น |

> **คำถามหลักที่ต้องตอบ:** ใครเป็นคนพาผู้เช่าไปดูห้อง/ปิดดีล? ถ้าเป็นทีมคุณเอง = แบบ A

## 2. กฎธุรกิจที่ต้องนิยามให้ชัด (ก่อนเขียนโค้ด)

1. **ฐานคิดค่านายหน้า** — ค่าเช่า 1 เดือนตามสัญญา (แนะนำ: ใช้ค่าเช่าห้องอย่างเดียว **ไม่รวม**ค่าน้ำ/ไฟ/ส่วนกลาง)
2. **ใครจ่าย** — มาตรฐานไทยคือ **เจ้าของหอจ่าย** (ไม่เก็บจากผู้เช่า → ผู้เช่าใช้ฟรี ทำให้ marketplace โตง่าย)
3. **เกิดสิทธิรับค่านายหน้าเมื่อไหร่** — แนะนำ: เมื่อ**เซ็นสัญญา + ผู้เช่าชำระเงินก้อนแรก** (มัดจำ/ค่าเช่าเดือนแรก) เพื่อกันเคลมลม
4. **Attribution window** — lead จาก Chao-Dee ต้องเซ็นสัญญาภายในกี่วันจึงนับ (แนะนำ **60–90 วัน**)
5. **ไม่คิดซ้ำ** — ผู้เช่าเดิมต่อสัญญา/ย้ายห้องในหอเดิม = ไม่คิดค่านายหน้าใหม่
6. **ยกเลิก/คืนเงิน** — ถ้าผู้เช่าออกภายใน X วัน (เช่น 30 วัน) คืนค่านายหน้ากี่ % (แนะนำมี ไม่งั้นเจ้าของหอไม่กล้าใช้)
7. **ภาษี** — ค่านายหน้าคือรายได้ค่าบริการ:
   - ถ้าจด VAT → +7% และต้องออกใบกำกับภาษี
   - เจ้าของหอที่เป็น**นิติบุคคล** จะ**หัก ณ ที่จ่าย 3%** (ภ.ง.ด.53) → ระบบต้องรองรับยอดรับจริงไม่เท่ายอดบิล
8. **การชำระ** — ออกใบแจ้งหนี้ค่านายหน้า → เจ้าของหอโอน+แนบสลิป → ทีมงานยืนยัน (แนวเดียวกับหน้าต่ออายุที่มีอยู่)

## 3. เอกสาร/กฎหมายที่ต้องมี 📄

| # | เอกสาร | หมายเหตุ |
|---|---|---|
| 1 | **สัญญา/ข้อตกลงนายหน้า** (Agency Agreement) กับเจ้าของหอ | ขอบเขตงาน · ค่าตอบแทน 1 เดือน · เงื่อนไขเกิดสิทธิ · attribution · การชำระ · ยกเลิก/คืนเงิน · ข้อพิพาท |
| 2 | **เพิ่มหมวดบริการเอเจนซี่ในข้อกำหนดการใช้งาน** (/terms) | อ้างอิงสัญญานายหน้า |
| 3 | **เพิ่มวัตถุประสงค์ DBD** ⚠️ | ต้องมี *"ประกอบกิจการนายหน้า/ตัวแทนจัดหาผู้เช่าอสังหาริมทรัพย์"* — ต่อจากเรื่องวัตถุประสงค์ที่ค้างอยู่ (ปัจจุบันมี 24 ข้อ ยังไม่มีข้อนายหน้า) |
| 4 | **ปรับ PDPA** (/privacy) | เพิ่มวัตถุประสงค์ "จัดหาผู้เช่า/นายหน้า" + การส่งข้อมูลผู้สนใจให้เจ้าของหอ |
| 5 | **นโยบายคืนเงิน/ข้อพิพาท** | ผู้เช่าออกก่อนกำหนด · ใครหามาก่อน |
| 6 | **เอกสารภาษี** | ใบแจ้งหนี้ค่านายหน้า · ใบเสร็จ/ใบกำกับภาษี · รองรับหนังสือรับรองหัก ณ ที่จ่าย 3% |
| 7 | *(ถ้าเลือกแบบ B)* สัญญากับเอเจนต์ + วิธีจ่ายส่วนแบ่ง + ภาษีเอเจนต์ | |

> **กฎหมายไทย:** นายหน้าอสังหาฯ **ไม่ต้องมีใบอนุญาตบังคับ** (มีเพียงใบรับรองแบบสมัครใจ) แต่ต้องมี **วัตถุประสงค์รองรับ** + เสียภาษีถูกต้อง — ให้ทนายยืนยันก่อนเปิดบริการ

## 4. โครงสร้างข้อมูล (migration ที่ต้องเพิ่ม)

```sql
-- ดีลนายหน้า
create table agency_deals (
  id uuid primary key,
  org_id uuid,              -- เจ้าของหอ
  listing_id uuid,          -- ประกาศต้นทาง
  lead_id uuid,             -- lead ต้นทาง (attribution)
  room_id uuid,
  tenant_id uuid,           -- ผู้เช่าที่ปิดได้
  contract_id uuid,         -- สัญญาที่เกิดขึ้น
  status text,              -- new | contacted | viewing | signed | invoiced | paid | cancelled | refunded
  rent_base numeric,        -- ค่าเช่า/เดือน ณ วันเซ็น
  commission_rate numeric,  -- 1.0 = 1 เดือน
  commission_amount numeric,
  signed_at, invoiced_at, paid_at, cancelled_at timestamptz,
  source text,              -- rent_marketplace | manual
  note text
);

-- ผูกสัญญากับ lead ต้นทาง (ใช้พิสูจน์ attribution)
alter table contracts add column lead_id uuid;

-- เจ้าของหอเปิดใช้บริการ + ยอมรับข้อตกลงเมื่อไหร่
alter table organizations
  add column agency_enabled boolean default false,
  add column agency_agreed_at timestamptz,
  add column agency_terms_version text;
```

## 5. ฟีเจอร์ที่ต้องสร้าง

### ฝั่งเจ้าของหอ (แอปจัดการ)
- เปิด/ปิดรับบริการเอเจนซี่ + **ยอมรับข้อตกลงนายหน้า** (เก็บ version + เวลา)
- หน้า "ดีลจากเอเจนซี่" — ดู lead ที่ส่งมา, สถานะ, ค่านายหน้าที่ต้องจ่าย
- ตอนสร้างสัญญา: เลือกว่า "ผู้เช่ารายนี้มาจาก Chao-Dee" → ผูก lead อัตโนมัติ
- ใบแจ้งหนี้ค่านายหน้า + แจ้งชำระ (แนบสลิป) — ใช้แพทเทิร์นเดียวกับหน้าต่ออายุ

### ฝั่งผู้เช่า (/rent)
- สื่อสารว่า **บริการหาห้องฟรีสำหรับผู้เช่า**
- ฟอร์ม "ให้เราช่วยหาห้อง" (งบ · ทำเล · จำนวนคน · วันที่เข้าอยู่)

### ฝั่ง Console (ทีมเอเจนซี่)
- **Pipeline ดีล** — new → ติดต่อแล้ว → นัดดู → เซ็น → วางบิล → เก็บเงิน
- ยืนยันการเซ็นสัญญา → คำนวณค่านายหน้าอัตโนมัติ
- ออกใบแจ้งหนี้ค่านายหน้า + ยืนยันการชำระ
- รายงาน: รายได้ค่านายหน้า · conversion rate · ค่าคอมค้างรับ

### อัตโนมัติ
- แจ้งเจ้าของหอเมื่อมี lead ใหม่ *(มีแล้ว)*
- แจ้งเตือนค่านายหน้าครบกำหนด
- เตือนดีลที่ค้างไม่อัปเดตสถานะ

## 6. ความเสี่ยงที่ต้องวางแผนรับมือ

| ความเสี่ยง | วิธีรับมือ |
|---|---|
| **เจ้าของหอตัดตอน (bypass)** — คุยกับผู้เช่าเองไม่แจ้ง | ข้อตกลงชัด + attribution window + ระบบเทียบผู้เช่าที่เพิ่มเข้ามากับ lead + บทลงโทษ/ระงับบริการ |
| **ข้อพิพาทว่าใครหามาก่อน** | log ครบ (เวลา lead, การติดต่อ, นัดดู) เป็นหลักฐาน |
| **ค่านายหน้าค้างชำระ** | มีระบบติดตาม + ระงับบริการเอเจนซี่ถ้าค้าง |
| **ผู้เช่าออกเร็ว** | นโยบายคืนเงินที่ชัด (เช่น ออกใน 30 วัน คืน 50%) |
| **หัก ณ ที่จ่าย 3%** | ระบบต้องบันทึกยอดบิล vs ยอดรับจริง + เก็บหนังสือรับรอง |

## 7. แผนเป็นเฟส

**Phase 0 — ตัดสินใจ + เอกสาร (1–2 สัปดาห์)** ← เริ่มที่นี่
- เลือกโมเดล (A/B/C) · นิยามกฎธุรกิจ 8 ข้อ · ร่างสัญญานายหน้า
- เพิ่มวัตถุประสงค์ DBD · ปรึกษาทนาย + บัญชี

**Phase 1 — MVP ใช้งานได้จริง**
- migration (agency_deals, contracts.lead_id, org.agency_*)
- Console: pipeline + ยืนยันเซ็น + คำนวณค่าคอม + ออกบิล
- เจ้าของหอ: ยอมรับข้อตกลง + ดูดีล + จ่ายค่าคอม

**Phase 2 — ขยาย**
- ฟอร์ม "ให้เราหาห้องให้" ฝั่งผู้เช่า · แจ้งเตือนอัตโนมัติ · รายงาน · ใบกำกับภาษีค่านายหน้า

**Phase 3 — เครือข่ายเอเจนต์ (ถ้าต้องการ)**
- สมัครเอเจนต์ · ส่วนแบ่ง · จ่ายเงิน + ภาษีเอเจนต์
